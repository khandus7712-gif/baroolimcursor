# Google OAuth 동의 화면 확인 가이드

## 🔴 "The OAuth client was not found" 오류의 실제 원인

브라우저 콘솔의 "경고!" 메시지는 **무시하세요**. 이것은 브라우저의 일반 보안 경고일 뿐입니다.

**실제 문제**: Google OAuth 동의 화면이 완료되지 않았을 가능성이 높습니다.

## ✅ Google Cloud Console에서 확인할 사항

### 1단계: OAuth 동의 화면 접속

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. **올바른 프로젝트 선택** (클라이언트 ID를 생성한 프로젝트)
3. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**

### 2단계: 동의 화면 상태 확인

다음 항목들이 **모두 설정**되어 있는지 확인:

#### 필수 항목:
- [ ] **사용자 유형**: 외부 또는 내부 선택되어 있는가?
- [ ] **앱 이름**: "바로올림" 또는 다른 이름이 입력되어 있는가?
- [ ] **사용자 지원 이메일**: 이메일이 입력되어 있는가?
- [ ] **개발자 연락처 정보**: 이메일이 입력되어 있는가?

### 3단계: 동의 화면 완료

만약 위 항목들이 비어있다면:

1. **사용자 유형** 선택 (외부 또는 내부)
2. **앱 이름** 입력: "바로올림"
3. **사용자 지원 이메일** 입력: 본인 이메일
4. **개발자 연락처 정보** 입력: 본인 이메일
5. **저장 후 계속** 클릭
6. **범위** 단계 → **저장 후 계속**
7. **테스트 사용자** 단계 (필요시) → **저장 후 계속**
8. **요약** 단계 → **대시보드로 돌아가기**

### 4단계: 앱 게시 상태 확인

**외부** 사용자 유형을 선택한 경우:

- **테스트 모드**: 최대 100명의 테스트 사용자만 로그인 가능
- **프로덕션**: 모든 사용자가 로그인 가능

개발 중이라면 **테스트 모드**로 유지하고, 테스트 사용자 목록에 본인 이메일을 추가하세요.

## 🔍 확인 체크리스트

- [ ] Google Cloud Console 접속
- [ ] 올바른 프로젝트 선택
- [ ] API 및 서비스 → OAuth 동의 화면 이동
- [ ] 사용자 유형이 선택되어 있는가?
- [ ] 앱 이름이 입력되어 있는가?
- [ ] 사용자 지원 이메일이 입력되어 있는가?
- [ ] 개발자 연락처 정보가 입력되어 있는가?
- [ ] 모든 단계가 완료되었는가?

## 💡 중요

**OAuth 동의 화면이 완료되지 않으면**, 클라이언트 ID가 있어도 `invalid_client` 오류가 발생합니다!

## 🚀 완료 후

1. OAuth 동의 화면 완료
2. 서버 재시작 (Ctrl+C 후 npm run dev)
3. Google 로그인 다시 시도
