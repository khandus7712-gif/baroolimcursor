# Google 로그인 "invalid_client" 오류 해결 가이드

## 🔴 현재 오류
- **오류 메시지**: "The OAuth client was not found"
- **오류 코드**: `401 오류: invalid_client`

## ✅ 확인된 사항
- ✅ 클라이언트 ID가 Google Cloud Console에 존재함
- ✅ 리디렉션 URI가 올바르게 설정됨
- ✅ 환경 변수가 올바르게 설정됨
- ✅ Secret의 마지막 4자리가 일치함

## 🔍 가장 가능성 높은 원인: OAuth 동의 화면 미완료

Google OAuth는 클라이언트 ID가 있어도 **OAuth 동의 화면이 완료되지 않으면** `invalid_client` 오류가 발생합니다.

## 📋 해결 방법

### 1단계: OAuth 동의 화면 확인

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. **올바른 프로젝트 선택** (클라이언트 ID를 생성한 프로젝트)
3. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**

### 2단계: 동의 화면 완료 확인

다음 항목들이 **모두 설정**되어 있는지 확인:

#### 필수 항목:
- [ ] **사용자 유형**: 외부 또는 내부 선택
- [ ] **앱 이름**: "바로올림" 또는 원하는 이름
- [ ] **사용자 지원 이메일**: 본인 이메일
- [ ] **개발자 연락처 정보**: 본인 이메일

#### 선택 항목 (권장):
- [ ] 앱 로고 (선택사항)
- [ ] 앱 도메인 (선택사항)
- [ ] 개인정보처리방침 URL (선택사항)
- [ ] 서비스 약관 URL (선택사항)

### 3단계: 동의 화면 저장

1. 모든 필수 항목 입력 후
2. **저장 후 계속** 버튼 클릭
3. **범위** 단계에서 **저장 후 계속**
4. **테스트 사용자** 단계에서 (필요시) 테스트 사용자 추가
5. **요약** 단계에서 **대시보드로 돌아가기** 클릭

### 4단계: 앱 게시 (프로덕션 사용 시)

**중요**: 외부 사용자 유형을 선택한 경우:

1. OAuth 동의 화면 페이지에서
2. **게시 상태** 확인
3. **앱 게시** 버튼 클릭 (프로덕션 사용 시)
4. 또는 **테스트 모드**로 유지 (개발 중)

**테스트 모드**에서는:
- 최대 100명의 테스트 사용자만 로그인 가능
- 테스트 사용자 목록에 이메일 추가 필요

### 5단계: 서버 재시작

OAuth 동의 화면을 완료한 후:

```bash
# 서버 중지 (Ctrl+C)
npm run dev
```

### 6단계: 다시 테스트

1. 브라우저에서 `http://localhost:3000/login` 접속
2. "Google로 시작하기" 버튼 클릭
3. 로그인 시도

## 🚨 여전히 오류가 발생하는 경우

### 추가 확인 사항

1. **올바른 프로젝트 사용 확인**
   - 클라이언트 ID가 생성된 프로젝트를 사용하고 있는지 확인
   - 프로젝트 선택기를 통해 확인

2. **클라이언트 ID 재확인**
   - Google Cloud Console → 사용자 인증 정보
   - 클라이언트 ID 목록에서 해당 ID 확인
   - 클릭하여 상세 정보 확인

3. **환경 변수 재확인**
   ```bash
   npm run check:env
   ```

4. **브라우저 캐시 클리어**
   - 개발자 도구 (F12) → 네트워크 탭
   - "Disable cache" 체크
   - 또는 시크릿 모드에서 테스트

5. **새 클라이언트 생성 (최후의 수단)**
   - 기존 클라이언트 삭제
   - 새 OAuth 클라이언트 ID 생성
   - OAuth 동의 화면 완료
   - `.env` 파일 업데이트
   - 서버 재시작
